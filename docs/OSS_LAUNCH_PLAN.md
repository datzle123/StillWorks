# OSS Launch Plan

## Positioning

Category: human-owned behavior contracts for CI.

Headline: **Show it once. Keep it working.**

AI is an amplifier, not the whole audience. Do not market MergeVow as another test generator,
pixel-diff tool, self-healing suite, or zero-flake system.

## Required Launch Assets

- `npx mergevow demo` completes without login or API key in under 60 seconds.
- A 40-second GIF shows a persistence regression and head-contract tampering.
- Three public demo PR sequences.
- A reproducible mutation benchmark.
- A static example failure report.
- Copy-paste local and GitHub Action quickstarts.
- Honest comparison with Playwright codegen plus CODEOWNERS.

## Release Sequence

- `0.1`: demo, record, semantic check, `--base`.
- `0.2`: pinned GitHub Action and semantic PR report.
- `0.3`: Protected Attestation only after fully bound approval, external enforcement, and isolated
  execution pass their demand and security gates.
- `0.4`: minimal same-origin fixture mode with redaction, only if demanded.

## Distribution

Quiet beta first, then Show HN, Playwright communities, GitHub Action Marketplace, relevant web and
open-source communities, and coding-agent communities. Use GitHub Discussions before opening a
separate Discord.

## Targets

| Horizon | Adoption target | Reach hypothesis |
|---|---|---|
| 30 days | 25 installs, 10 weekly-active repos, 3 contributors | 250-500 stars |
| 90 days | 100 installs, 40 weekly-active repos, 3 case studies | 750-1,500 stars |
| 180 days | 300 installs, 120 weekly-active repos, 10 regression stories | 2,000-4,000 stars |

Stars are not the north star. If stars grow while fewer than 20 repositories are weekly active at day
90, stop feature expansion and fix retention.
